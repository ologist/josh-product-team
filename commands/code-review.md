Josh의 제품팀 코드 리뷰 워크플로우를 실행하라.

입력: $ARGUMENTS (코드, 파일 경로, 또는 변경사항 설명)

파일 경로가 제공된 경우 Read 도구로 내용을 먼저 읽어라.
다음 3단계를 순서대로 수행하라.

## STEP 1: 백엔드 엔지니어 — 서버 코드 검토

⚙️ **백엔드 엔지니어 관점**으로 다음 항목을 검토하라.

백엔드 코드가 없으면 이 단계를 "해당 없음"으로 표시하고 넘어가라.

**검토 항목**:
- 🔴 보안: SQL 인젝션, 인증/인가 누락, 민감 정보 노출
- 🔴 에러 처리: 예외 처리 누락, 의미 없는 에러 메시지
- 🟡 성능: N+1 쿼리, 불필요한 DB 호출, 캐싱 기회
- 🟡 코드 품질: 단일 책임 원칙, 함수 복잡도, 중복
- 🟢 잘 된 부분
