# Josh Product Team Plugin

PM · 백엔드 · 프론트엔드 3인의 AI 에이전트가 협력하여 기획부터 구현까지 지원하는 Claude 플러그인.

## 에이전트 구성

| 에이전트 | 역할 | 활성화 방법 |
|---------|------|-----------|
| 🗂️ PM | PRD 작성, 유저 스토리, 우선순위 결정 | "PM으로서 기획해줘", "PRD 작성해줘", "요구사항 정리해줘" |
| ⚙️ 백엔드 엔지니어 | API 설계, DB 스키마, 아키텍처, 코드 리뷰 | "API 설계해줘", "DB 스키마 짜줘", "백엔드 코드 리뷰해줘" |
| 🎨 프론트엔드 엔지니어 | 컴포넌트 설계, UI/UX, 상태 관리, 성능 | "컴포넌트 구조 잡아줘", "UI 설계해줘", "프론트엔드 코드 리뷰해줘" |

## 워크플로우 커맨드

| 커맨드 | 설명 |
|--------|------|
| `/feature-plan` | 기능 기획 — PM PRD → 백엔드 API 스펙 → 프론트엔드 UI 스펙 → 실행 계획 |
| `/code-review` | 코드 리뷰 — 백엔드·프론트엔드 코드 검토 + PM 요구사항 부합 확인 |
| `/sprint-plan` | 스프린트 계획 — PM 우선순위 → 엔지니어 공수 산정 → 최종 백로그 |

## 개별 에이전트 단독 호출 예시

```
# PM
"이 기능의 PRD를 작성해줘"
"유저 스토리 형식으로 요구사항 정리해줘"
"기능 A, B, C 중 어떤 걸 먼저 만들어야 할까? 우선순위 정해줘"

# 백엔드 엔지니어
"사용자 인증 API를 설계해줘"
"이 쿼리 성능이 걱정되는데 개선 방법 알려줘"
"이 백엔드 코드 리뷰해줘"

# 프론트엔드 엔지니어
"이 화면의 컴포넌트 구조를 잡아줘"
"무한 스크롤 어떻게 구현하면 좋을까"
"이 React 코드 리뷰해줘"
```

## 설계 원칙

- **실행 가능한 산출물**: 추상적 조언보다 바로 사용 가능한 스펙·코드·스키마 제공
- **에이전트 간 협업**: 워크플로우 커맨드에서 각 에이전트가 서로의 결과물을 인풋으로 활용
- **Human-in-the-Loop**: 최종 결정은 항상 Josh가 내림
